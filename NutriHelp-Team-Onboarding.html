NutriHelp Mobile App — Team Onboarding Guide

Project: NutriHelp-App-2026  |  Platform: Android (Expo / React Native)  |  Updated: April 2026

1. What Is This Project?

NutriHelp is a cross-platform mobile application that helps users track nutrition, scan food barcodes, discover recipes, and manage health and wellness goals. It is built with Expo SDK 54 (React Native + TypeScript) and connects to a hosted backend API.

ResourceURL
GitHub Repositoryhttps://github.com/Gopher-Industries/NutriHelp-App-2026
Backend APIhttps://nutrihelp-backend-deployment.onrender.com
API Docs (Swagger)https://nutrihelp-backend-deployment.onrender.com/api-docs
Note: The backend is hosted on Render's free tier. The first API request after inactivity may take ~30 seconds to wake the server.

Set Up Android Emulator (Android Studio)

  1. Open Android Studio
  2. Go to Tools → Device Manager → Create Device
  3. Select Phone → Pixel 6 or Pixel 7
  4. Select API Level 33 (download if prompted)
  5. Click Finish
  6. Press the ▶ Play button to start the emulator

3. Getting the Project Running Locally

Step 1 — Clone the Repository

git clone https://github.com/Gopher-Industries/NutriHelp-App-2026.git
  cd NutriHelp-App-2026

Step 2 — Install Dependencies

npm install
  npx expo install

npm install handles all JavaScript packages. npx expo install ensures all Expo SDK packages are at the correct compatible versions.

Step 3 — Set Up Environment Variables

cp .env.example .env

The .env.example file already contains the backend API URL and Supabase credentials. Simply copy it — no manual changes needed for most developers.

Important: Never commit the .env file to Git. It is already listed in .gitignore.

Step 4 — Start the Development Server

npx expo start --clear

Then choose how to run the app:

  • Press a → opens on Android emulator (must be running first)
  • Scan the QR code with Expo Go app on your phone (same WiFi required)

5. Branch Strategy

BranchPurpose
mainProduction-ready. Protected — PRs required, 1 approval minimum, no direct pushes.
feature/M-XX-nameIndividual feature work. Example: feature/M-13-login-screen

Daily Workflow

# 1. Always start from main
  git checkout main
  git pull origin main

  # 2. Create your feature branch (use your task number)
  git checkout -b feature/M-XX-your-feature-name

  # 3. Write code, commit often
  git add .
  git commit -m "feat: description of what you did"

  # 4. Push your branch
  git push origin feature/M-XX-your-feature-name

  # 5. Open a Pull Request into main on GitHub
  # 6. Get 1 approval → merge
Rule: Never push directly to main. Always use a feature branch and open a PR — merges require at least 1 approval.

8. Testing on a Physical Android Device

For large projects, Expo Go can be slow. A development build APK is faster and supports all native features.

  1. Install EAS CLI: npm install -g eas-cli
  2. Login: eas login
  3. Build the APK: eas build --platform android --profile development --local
  4. Install the generated .apk on your Android phone
  5. Run: npx expo start --dev-client
  6. Connect your phone — hot reload works instantly
Tip: The dev APK only needs to be rebuilt when native dependencies change. JS/UI changes always hot-reload.
